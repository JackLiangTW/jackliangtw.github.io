跳轉機 / 堡壘機操作筆記與 MongoDB 備份流程

進場方式(通用流程)

  1. 公司 VPN 連上。
  2. Web UI(JumpServer / Teleport 等)→ 我的資產 → 找到目標主機。
  3. Web Terminal 直接拉 SSH;或從 UI 下載臨時 SSH config 後用本機 terminal 連。

跳轉機會記錄所有操作,不要在上面跑會造成不可逆破壞的指令;有疑慮先在 staging 測。

進 container 看服務

# 主機上
docker ps                                # 找目標 container id / name
docker exec -it <id> bash                # 進去

# 內部
pm2 list
pm2 logs <app_name> --lines 200
pm2 logs --nostream --lines 100          # 不 tail,直接印最新

# 退出
exit

MongoDB(docker 容器內)

docker exec -it <mongo-container> mongosh

use admin
db.auth("admin", "__MONGO_PASSWORD__")

use myDB
db.employees.find({ name: "naomi" })
db.open_code_240116.find({ lotteryId: "251" }).sort({ expect: -1 }).limit(5)

db.applications.find({
  employeeId: "...",
  leaveType: "...",
  remark: "..."
})

db.collection.updateOne({ _id: ObjectId("...") }, { $set: { "nested.field": v } })
db.collection.deleteOne({ _id: ObjectId("...") })

Mongodump 匯出(跳轉機場景)

CID=<mongo-container>

# 1) 在 container 內 dump 到 /dump
docker exec -i $CID mongodump \
  --username admin --password __MONGO_PASSWORD__ \
  --authenticationDatabase admin \
  --db myDB --out /dump

# 2) 把 dump 拷貝出來
docker cp $CID:/dump ./dump

# 3) 壓縮方便下載
tar -czvf backup.tar.gz ./dump

下載回本機:

  • JumpServer UI 檔案管理器 → 對應主機目錄 → 下載 .tar.gz。
  • 或用 sftp / scp(若跳轉機允許)。

Bson → JSON 批次:

for f in dump/myDB/*.bson; do
  bsondump "$f" > "${f%.bson}.json"
done

SSH ProxyCommand(本機直連跳板後的機器)

有時候跳板會允許你配 ~/.ssh/config,本機直接登入內網機:

Host jump
  HostName __JUMP_HOST__
  User your-user
  IdentityFile ~/.ssh/id_rsa_jump

Host dev-mars
  HostName 192.168.168.171
  User root
  ProxyJump jump

之後 ssh dev-mars 就會先走 jump 再到目的主機;scp dev-mars:/path/ . 也能用。

注意

  • 線上 DB 任何 updateMany / deleteMany 前一定先 find 看會命中幾筆,再送改動。
  • mongodump 會鎖住 oplog 尾端的索引,對線上大資料表可能造成短暫 IO 壓力,建議在離峰時段。
  • 跳轉機通常會 session timeout;長時間跑的 command 用 nohup 或 tmux / screen 包起來。
