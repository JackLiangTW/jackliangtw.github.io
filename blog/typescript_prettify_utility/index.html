TypeScript Prettify Utility Type:讓 IDE hover 顯示清楚

Prettify<T> — 讓 IDE hover 展開合併後型別

複雜的 intersection 型別在 VSCode / WebStorm 上 hover 時往往顯示成 A & B & C,看不清楚實際 shape。用這個 utility 把它強制展開成單一 object:

type Prettify<T> = { [K in keyof T]: T[K] } & {};

用法:

type Raw = { a: string } & { b: number } & { c: boolean };
type Clean = Prettify<Raw>;
//   ↑ hover 顯示:{ a: string; b: number; c: boolean }

原理:

  • { [K in keyof T]: T[K] } 把所有 key 重新映射一次,TS 就得把 intersection 展平。
  • 最後的 & {} 是雞肋但必要的 trick,強迫 TS 再做一次合併輸出。

實用場景:

  • 寫給他人用的 public API 型別(尤其是 library)。
  • 有多層 Omit / Pick / 條件型別的結果。
  • 型別命名為 T、U 看不出結構,幫使用者 hover 時直接看到。

其他常用 utility 複習

type ReadonlyDeep<T> = { readonly [K in keyof T]: ReadonlyDeep<T[K]> };
type NonNullableDeep<T> = { [K in keyof T]: NonNullable<T[K]> };
type Awaited<T> = T extends Promise<infer U> ? Awaited<U> : T;

記得 TS 4.5+ 內建 Awaited<T>。
