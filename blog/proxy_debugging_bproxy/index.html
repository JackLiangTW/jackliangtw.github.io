封包除錯:bproxy 設定、adb proxy 與 HTTPS 擷取流程

bproxy(本機代理工具)

npm install -g bproxy
bproxy start                  # 啟動
  • 面板:http://localhost:8888/web/
  • 憑證下載:http://localhost:8888/install(裝置安裝後才能看 HTTPS 明文)

macOS Safari 設定

系統設定 → Wi-Fi → 詳細資訊 → Proxies →

  • 自動代理伺服器 URL,或
  • 手動 HTTP / HTTPS 各填 127.0.0.1:8888

Chrome 踩過的雷

Chrome 會拒絕 bproxy 自簽根憑證,跳 NET::ERR_CERT_AUTHORITY_INVALID,改用 Safari / Firefox 測試即可。

非瀏覽器 App(Line / Telegram)

  • bproxy 需要系統級信任或 root 才攔得到 App 流量。
  • iOS 要安裝描述檔 + 手動開啟「完全信任」;Android 要把 CA 移到 system cert 或用 Magisk 模組。

Android 模擬器 / 裝置的 adb proxy

# 設定系統全域 proxy(Wi-Fi settings 亂改會被重置,adb 設的比較穩)
adb shell settings put global http_proxy __HOST__:8888

# 動態抓本機 IP(macOS)
adb shell settings put global http_proxy $(ipconfig getifaddr en0):8888

# 取消
adb shell settings put global http_proxy :0

常見坑:

  • AVD 內建 proxy 設定只影響 Chrome / WebView,其他 App(含 Line)會直接沒網路。用 adb shell settings put global http_proxy 才會全域套用。
  • 設好 proxy 後若 App 用 Certificate Pinning,依然看不到明文 → 需要 Frida 或 Magisk 模組繞過。

Maya(測試環境筆記)

  • MacOS AVD:Pixel API 32(Android 12)或 API 27(Android 8.1)。
  • Genymotion:Xiaomi API 30(Android 11)。
  • 不同 API 版本行為差異:API 24 以上 App 預設不信任 user CA,要改 network security config。

擷取 HTTPS 的整體流程(建議)

  1. 起 bproxy / mitmproxy / Charles。
  2. 裝置安裝對應 root CA,並移到 System 層或改 App 的 network security config。
  3. 若有 SSL Pinning → Frida hook 或 JustTrustMe Xposed 模組。
  4. 若 App 用 gRPC / 自家 TLS → 可能要搭配反編譯找到 pin 位置手動改 smali。
