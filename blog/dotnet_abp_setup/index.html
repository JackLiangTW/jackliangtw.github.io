.NET + ABP.io:macOS 安裝、UnitOfWork、EF Migration 與 OpenIddict 繞過

環境安裝(macOS ARM64)

# 1) 裝 .NET 8 SDK(官網下載 macOS arm64 pkg)
#    https://dotnet.microsoft.com/en-us/download/dotnet/8.0

# 2) PATH
echo 'export DOTNET_ROOT=$HOME/dotnet' >> ~/.zshrc
echo 'export PATH=$PATH:$HOME/dotnet:$HOME/.dotnet/tools' >> ~/.zshrc

# 3) ABP CLI(版本要和專案相同)
dotnet tool install -g Volo.Abp.Cli --version 7.2.3

# 4) EF CLI
dotnet tool install --global dotnet-ef

常見卡點:

  • abp: command not found → ~/.dotnet/tools 沒進 PATH。
  • Source Generators 在 .NET 7 macOS 會偶爾炸掉 → 升 .NET 8。
  • Rider 把 obj/ 鎖住無法寫 → 關掉 Rider 重開,必要時 sudo chmod -R 777 obj bin。

專案啟動(ABP Web + DbMigrator)

  1. phpMyAdmin 建 DB:UTF8MB4(utf8mb4_0900_ai_ci),再建 hangfire DB。
  2. 新建 admin 帳號開 GRANT ALL。
  3. cd src/Mars.Web && abp install-libs && yarn build-css。
  4. 跑 Mars.DbMigrator → 跑 Mars.Web。

UnitOfWork

using var uow = _unitOfWorkManager.Begin();
await _repo1.InsertAsync(a);
await _repo2.UpdateAsync(b);
await uow.CompleteAsync(cancellationToken);      // 沒叫 Complete 就視同回滾

重點:

  • using var 語法:離開 scope 若沒 CompleteAsync → 整個 transaction rollback。
  • UoW 管理資料庫層級的鎖,跨多個 repository 的原子性由它保證。
  • 想做跨 DB 分散式交易要另外啟用 TCC / Saga,ABP 預設只支援單一 DbContext。

ISoftDelete vs 硬刪除

操作 對帶 ISoftDelete 的實體行為
DeleteAsync(entity) 只把 IsDeleted = true,不串到子表
DeleteDirectAsync(entity) 真正 SQL DELETE,會觸發 EF 的 DeleteBehavior.Cascade

子表想跟著軟刪除 → 要自己寫 logic,不會自動傳染。

EF Migration 差異比對

  • dotnet ef migrations add <name> → 比較最新 snapshot 與現在 DbContext,產生 Up/Down SQL。
  • 衝突(多人同時改 model):刪掉自己那份 migration、pull → rebase → 重新 add。
  • dotnet clean 清 obj/bin;Rider 快取怪怪的就重啟 Rider。

OpenIddict 憑證過期繞過(僅 dev 環境)

ABP 專案起不來、錯誤是 OpenIddict 憑證過期:關掉自動產憑證、改塞 ephemeral key。

// Module 的 PreConfigureServices
PreConfigure<AbpOpenIddictAspNetCoreOptions>(options =>
{
    options.AddDevelopmentEncryptionAndSigningCertificate = false;
});

PreConfigure<OpenIddictServerBuilder>(builder =>
{
    if (hostingEnvironment.IsDevelopment())
    {
        builder
            .AddEphemeralEncryptionKey()
            .AddEphemeralSigningKey();
    }
});
  • Ephemeral key 只在記憶體,重啟後重新產生 → 不會過期。
  • 別用在正式環境。正式環境要有穩定的 signing cert(檔案或憑證庫)。
  • validation options 與其他 AddServer 設定保留,只替換 key 來源。
